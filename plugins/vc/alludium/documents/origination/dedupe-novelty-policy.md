# Dedupe And Novelty Policy

## Match Signals

- Company domain
- Legal name and aliases
- Founder names
- CRM record URLs
- Source object IDs
- Existing Deal Room fields
- Prior rejected or watched candidates
- Relationship context

## Outcomes

- New: no meaningful match found.
- Existing opportunity: attach evidence to the current Deal Room recommendation.
- Existing pipeline candidate: update the candidate batch or source registry.
- Prior reject or watch: require partner review before promotion.

## Standard

Novelty is a decision input, not a guarantee. When uncertain, mark ambiguous and include the competing matches.
