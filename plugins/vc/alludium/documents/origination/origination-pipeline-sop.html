Origination Pipeline Operating SOP

Operating method for standing venture sourcing pipelines.

When this source is used as a rendered task output, create the final deliverable as a standalone safe static HTML artifact with a .html filename and mimeType: "text/html". Use the shared VC document styling rules from Template Use Guidance and make any generated links open in a new tab or window with target="_blank" and rel="noopener noreferrer". Do not create Markdown output artifacts from this source.

Purpose

Use this SOP to operate a standing sourcing machine. Origination finds, enriches, dedupes, scores, and promotes candidates. Deal Pipeline work starts only after a candidate is promoted or a partner explicitly creates an opportunity workspace.

Workflow

StepOutputApproval Gate
Configure thesis, geography, stage, source scope, review policy, budgets, and source taxonomySource registryHuman confirms scope
Run discovery and read-preview tasksSetup and preview artifacts with source provenanceHuman approves source readiness
Batch candidates for enrichment, dedupe, scoring, and relationship checksCandidate batch with source family, freshness, and evidence qualityHuman reviews batch quality
Produce digests and outreach queuesSourcing digest and outreach queueHuman approves outreach or follow-up
Promote qualified candidatesPromotion package with thesis rationale, source path, screening result, risks, and recommended Deal Pipeline postureHuman approves Deal Pipeline creation or update

Source Provenance

Every candidate should preserve source family, source detail, owner, freshness, relationship path, thesis rationale, evidence quality, dedupe state, and promotion rationale. Promotion should carry that trail into the downstream Deal Pipeline.

Controls

Do not create Deal Pipelines, mutate CRM records, contact founders, or enable recurring schedules without explicit human approval.
