# Technical Evaluation Guide

## Purpose

Use this guide to evaluate technical plausibility and defensibility before formal diligence. The output should identify the technical claims that matter most to the investment case and what evidence is needed to verify them later.

## Core Questions

| Area | Evaluation Question | Evidence To Seek |
| --- | --- | --- |
| Product | What has been built and what is still roadmap? | Demo notes, product screenshots, deck, release notes, customer usage. |
| Technical edge | What is the claimed technical advantage? | Architecture notes, model/data claims, benchmarks, patents, founder explanation. |
| Architecture | Does the technical approach seem plausible and scalable? | System overview, infrastructure notes, integration complexity, team explanation. |
| Data and AI | Are data sources, model claims, latency, rights, and evaluation methods credible? | Data-source summary, model results, validation method, compliance notes. |
| IP and defensibility | Is the edge protected by IP, data, process, switching cost, or know-how? | Patent claims, trade-secret posture, assignments, proprietary data, workflow lock-in. |
| Roadmap | Can the roadmap support commercial milestones? | Milestone plan, engineering capacity, dependencies, product gaps. |
| Team coverage | Does the team have enough technical leadership? | Founder/team bios, CTO/technical lead role, hiring gaps. |

## Output Standard

| Section | Required Content |
| --- | --- |
| Current technical view | Positive / mixed / negative / unknown. |
| Main technical thesis | The technical belief that must be true. |
| Technical claims register | Claim, evidence, confidence, and proof gap. |
| Gating risks | Risks that could block decision review or formal diligence. |
| Next proof needed | Demo, architecture review, expert call, model/data evidence, IP documents, or code-access plan. |
| Decision impact | Whether the opportunity should continue, pause, or require targeted technical diligence. |

## Boundary

Technical evaluation is not a code review, security audit, expert technical diligence, or IP clearance. State when those inputs are still required.
