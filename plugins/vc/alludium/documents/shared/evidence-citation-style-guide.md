# Evidence And Citation Style Guide

## Purpose

Use this guide whenever a VC task produces a memo, scorecard, digest, diligence report, decision record, promotion package, or review artifact. The document should make it easy for an investment team to distinguish evidence, inference, assumption, stale information, and open risk.

## Citation Rules

| Claim Type | Required Treatment |
| --- | --- |
| Numeric claim | Cite source, period, units, and artifact or URL. Recompute derived values if used in a recommendation. |
| Founder or management claim | Label as founder-provided unless independently verified. |
| Third-party evidence | Cite source name, date, and link or artifact ID. Note source quality if weak. |
| Inference | Use "evidence suggests" or "we infer" and name the evidence base. |
| Assumption | Label as assumption and state what would confirm or falsify it. |
| Stale source | Keep visible with date and explain whether it is still usable. |

## Preferred Language

Use concise investment language. Prefer "evidence suggests" over "proves" unless the source is definitive. Use "not yet validated" for gaps. Use "requires partner judgment" when a decision depends on fund strategy rather than task evidence.

## Source Index Standard

Every durable output should end with a compact source index when material evidence is used.

| Ref | Source | Date / Period | Used For | Confidence | Link or Artifact |
| --- | --- | --- | --- | --- | --- |
| S1 | Source name | Date | Claim or section | High / Medium / Low | URL or artifact ID |
